# Case Study: Settings Page Toggles

## Context

- **Product**: Productivity app (Todoist-style)
- **Page/Screen**: Settings → Notifications panel
- **Target Audience**: General users managing their preferences
- **Agent(s) Used**: Microcopy Review Agent

## Before (Original Content)

> - "Enable or disable email notifications for when tasks are assigned to you"
> - "Toggle on/off push notifications to your mobile device for upcoming deadline reminders"
> - "Activate weekly digest email summary of all project activity and updates"

## After (Agency Output)

> **Analysis:** Toggle labels describe the mechanism ("Enable or disable", "Toggle on/off", "Activate") instead of the outcome. Users want to know *what they'll get*, not *how the toggle works*. Inconsistent phrasing across all three.
>
> **Suggestions:**
> - "Task assignments" — Description: "Get emailed when someone assigns you a task"
> - "Deadline reminders" — Description: "Push notification before a task is due"
> - "Weekly digest" — Description: "Email summary of project activity every Monday"
>
> **Rationale:** Toggle labels should name the notification type (noun), not describe the toggle action (verb). The description below explains what happens when it's on. Added "every Monday" to the digest for specificity. All three now follow the same pattern: short label + one-sentence description.

## What Changed & Why

| Change | Before | After | Principle |
|--------|--------|-------|-----------|
| Label pattern | Verb phrases ("Enable or disable...") | Noun labels ("Task assignments") | Toggles are on/off — the label names what's toggled |
| Removed mechanism language | "Toggle on/off", "Activate" | Removed | Users know how toggles work |
| Added specificity | "weekly digest email summary" | "every Monday" | When exactly? Be specific |
| Consistent structure | 3 different patterns | All: noun label + description sentence | Visual and cognitive consistency |
| Channel clarity | Buried in text | Stated in description ("emailed", "push notification", "email summary") | Users know which channel each notification uses |

## Measurable Difference

| Metric | Before | After |
|--------|--------|-------|
| Avg label length | 62 chars | 16 chars |
| Character reduction | — | 74% |
| Consistent pattern | No (3 different) | Yes (noun + description) |
| Channel identified | Buried in text | Explicit in each |
| Scannability | Low (must read full sentence) | High (scan noun labels) |
