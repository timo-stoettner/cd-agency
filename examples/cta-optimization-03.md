# Case Study: Developer Documentation API Key Signup

## Context

- **Product**: API platform (Stripe-style developer docs)
- **Page/Screen**: API documentation landing page — get API key
- **Target Audience**: Developers evaluating the API for integration
- **Agent(s) Used**: CTA Optimization Specialist

## Before (Original Content)

> Button: "Sign Up"
> Below: "Create an account to get started."

## After (Agency Output)

> **CTA Variations:**
> 1. **"Get your API key"** [RECOMMENDED] — *Outcome-focused*: Developers don't want to "sign up," they want an API key
> 2. **"Start building"** — *Action visualization*: Skips the admin step, focuses on what they'll do with the key
> 3. **"Try the API free"** — *Risk removal*: "Free" + "try" reduces commitment perception
> 4. **"See it in action — live sandbox"** — *Proof before commitment*: Let them test before signing up
> 5. **"Get started in 5 minutes"** — *Time specificity*: Developers value knowing the time cost upfront
>
> **Pre-CTA headline:** "Build your first integration in minutes"
> **Post-CTA reassurance:** "Free tier: 1,000 API calls/month. No credit card required."
>
> **A/B test plan:** Test #1 vs #4. Hypothesis: Developers prefer to evaluate before committing — a sandbox (#4) will attract 2x more engaged signups than direct API key access (#1), even if raw conversion is lower.

## What Changed & Why

| Change | Before | After | Principle |
|--------|--------|-------|-----------|
| Generic "Sign Up" | "Sign Up" | "Get your API key" | Name the actual thing they get, not the process |
| Vague supporting text | "Create an account to get started" | "Free tier: 1,000 calls/month. No credit card." | Developers want specifics: what's free, what are the limits |
| No sandbox option | None | "Live sandbox" variation | Developers want to test before committing to signup |
| No time commitment info | None | "in 5 minutes" variation | Time is the developer's scarcest resource |
| No usage context | None | "Build your first integration" headline | Show what they'll achieve, not what they'll fill out |

## Measurable Difference

| Metric | Before | After |
|--------|--------|-------|
| CTA specificity | Generic ("Sign Up") | Outcome-specific ("Get your API key") |
| Objections addressed | 0 | 3 (cost, time, commitment) |
| Developer empathy | Low (standard signup pattern) | High (API key, sandbox, time estimate) |
| Supporting details | 6 vague words | Specific tier + limits + no CC |
| Testable variations | 1 | 5 with hypotheses |
