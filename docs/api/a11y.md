# `tools.a11y_checker` — Accessibility Checker

> Stability: Stable

```python
from tools.a11y_checker import A11yChecker, A11yResult, A11yIssue, A11ySeverity
```

WCAG text-level accessibility checks. No API key required.

## Enum: `A11ySeverity`

```python
class A11ySeverity(str, Enum):
    CRITICAL = "critical"
    HIGH = "high"
    MEDIUM = "medium"
    LOW = "low"
```

## Class: `A11yIssue`

A single accessibility issue.

```python
@dataclass
class A11yIssue:
    rule: str
    severity: A11ySeverity
    message: str
    wcag_criterion: str
    suggestion: str = ""
    matches: list[str] = field(default_factory=list)
```

### `issue.to_dict()`

- Returns: `dict`

---

## Class: `A11yResult`

Result of accessibility content audit.

```python
@dataclass
class A11yResult:
    issues: list[A11yIssue]
    reading_grade: float
    target_grade: float
```

### `result.passed`

`True` if no critical or high severity issues.

- Type: `bool` (property)

### `result.critical_issues`

Issues with `CRITICAL` severity.

- Type: `list[A11yIssue]` (property)

### `result.high_issues`

Issues with `HIGH` severity.

- Type: `list[A11yIssue]` (property)

### `result.issue_count`

Total number of issues.

- Type: `int` (property)

### `result.label`

Human-readable status label.

- Type: `str` (property)

| Condition | Label |
| --- | --- |
| No issues | `"Pass"` |
| Critical issues | `"Fail (critical issues)"` |
| High issues | `"Fail (high severity issues)"` |
| Medium/low only | `"Pass (with warnings)"` |

### `result.to_dict()`

- Returns: `dict`

---

## Class: `A11yChecker`

### Constructor

```python
A11yChecker(
    target_grade: float = 8.0,
    max_emoji_per_100_words: int = 3,
)
```

- `target_grade`: Maximum acceptable Flesch-Kincaid grade level.
- `max_emoji_per_100_words`: Maximum emoji density.

### `checker.check(text)`

Run all accessibility checks on text.

- `text`: `str`
- Returns: `A11yResult`

```python
checker = A11yChecker(target_grade=8.0)
result = checker.check("Click here to learn more about our features")

print(f"Passed: {result.passed}")
print(f"Issues: {result.issue_count}")
for issue in result.issues:
    print(f"  [{issue.severity.value}] {issue.rule}: {issue.message}")
    print(f"    WCAG: {issue.wcag_criterion}")
```

## Accessibility Rules

### `reading-level` (high) — WCAG 3.1.5

Flags text whose Flesch-Kincaid grade level exceeds the target (default: 8.0).

### `sentence-length` (medium) — WCAG 3.1.5

Flags sentences over 25 words. Recommendation: aim for 15–20 words per
sentence.

### `no-all-caps` (medium) — WCAG 1.3.1

Flags ALL CAPS text that isn't a known abbreviation. Screen readers may spell
these letter by letter.

**Allowed abbreviations:** `API`, `URL`, `HTML`, `CSS`, `JS`, `JSON`, `XML`,
`SQL`, `HTTP`, `HTTPS`, `REST`, `SDK`, `CLI`, `UI`, `UX`, `ID`, `FAQ`, `PDF`,
`CSV`, `AWS`, `GCP`, `CI`, `CD`, `AM`, `PM`, `EST`, `PST`, `UTC`, `WCAG`,
`ARIA`, `OK`, `US`, `EU`, `UK`.

### `emoji-overuse` (medium) — WCAG 1.1.1

Flags excessive emoji usage (> 3 per 100 words). Screen readers announce each
emoji individually.

### `descriptive-link-text` (high) — WCAG 2.4.4

Detects "click here" anti-patterns:

- "click here"
- "tap here"
- "press here"
- "here to..."
- "click this"
- "click the link"
- "follow this link"

### `image-alt-text` (critical) — WCAG 1.1.1

Detects markdown images without alt text: `![](url)`.

### `meaningful-alt-text` (high) — WCAG 1.1.1

Detects images with filenames as alt text: `![image-123.png](url)`.
